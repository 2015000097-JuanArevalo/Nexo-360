# 04 - Plan de Demo y Checklist

## Objetivo

Preparar una demostración sencilla y clara de NEXO 360.

La demo debe mostrar que la plataforma funciona y que resuelve un problema real de organización escolar y de eventos.

---

## Duración recomendada

La demostración puede durar entre **5 y 8 minutos**.

Debe ser directa, sin explicar detalles técnicos innecesarios.

---

## Guion simple para la presentación

## 1. Presentar el problema

Explicar brevemente que muchas actividades escolares se organizan con mensajes, papeles o archivos separados.

Ejemplo:

> Muchas veces la información de tareas, permisos, eventos y comunicados está separada en diferentes medios. Esto causa confusión, pérdida de datos y más trabajo para estudiantes, docentes y organizadores.

---

## 2. Presentar la solución

Explicar qué es NEXO 360.

Ejemplo:

> NEXO 360 es una plataforma que reúne en un solo lugar la información escolar, los permisos, los eventos y las herramientas de organización.

---

## 3. Mostrar la página web pública

Acciones para la demo:

- Abrir la página web.
- Mostrar información de un evento.
- Llenar un formulario de registro.
- Mostrar el mensaje de confirmación.

Objetivo de esta parte:

- Demostrar que personas externas pueden ver información y registrarse fácilmente.

---

## 4. Iniciar sesión como estudiante

Acciones para la demo:

- Entrar con una cuenta de estudiante.
- Mostrar el panel principal.
- Abrir tareas.
- Abrir anuncios.
- Mostrar notas o asistencia.
- Mostrar un permiso asignado.

Objetivo de esta parte:

- Enseñar cómo el estudiante consulta su información escolar.

---

## 5. Iniciar sesión como docente

Acciones para la demo:

- Entrar con una cuenta de docente.
- Mostrar el panel del docente.
- Crear o mostrar una tarea.
- Publicar o mostrar un anuncio.
- Mostrar archivos o información de clase.

Objetivo de esta parte:

- Enseñar cómo el docente comparte información con estudiantes.

---

## 6. Iniciar sesión como administrativo

Acciones para la demo:

- Entrar con una cuenta administrativa.
- Mostrar el módulo de permisos.
- Generar o mostrar un permiso con código QR.
- Explicar que el QR ayuda a validar permisos de forma más segura.

Objetivo de esta parte:

- Demostrar el control de permisos dentro de la plataforma.

---

## 7. Iniciar sesión como organizador

Acciones para la demo:

- Entrar con una cuenta de organizador.
- Revisar lista de participantes.
- Validar un código QR.
- Mostrar inventario o mapa si está listo.
- Mostrar mensajes o avisos de organización.

Objetivo de esta parte:

- Enseñar cómo la plataforma ayuda en eventos y actividades.

---

## 8. Cierre de la demo

Explicar que esta es la primera versión y que puede crecer en el futuro.

Ejemplo:

> Esta primera versión demuestra las funciones principales. En una siguiente etapa, NEXO 360 podría incluir notificaciones, reportes avanzados, chat en tiempo real, portal para padres y más herramientas para colegios y eventos.

---

# Checklist antes de entregar

## Funciones principales

- [ ] La aplicación abre correctamente.
- [ ] El inicio de sesión funciona.
- [ ] Cada usuario entra a su panel correspondiente.
- [ ] El estudiante puede ver información escolar.
- [ ] El docente puede ver o publicar contenido.
- [ ] El administrativo puede revisar o generar permisos.
- [ ] El organizador puede revisar participantes.
- [ ] La página web pública muestra información del evento.
- [ ] El formulario de registro funciona.
- [ ] El código QR se puede mostrar o validar.

---

## Datos de prueba

- [ ] Existe una cuenta de estudiante.
- [ ] Existe una cuenta de docente.
- [ ] Existe una cuenta administrativa.
- [ ] Existe una cuenta de organizador.
- [ ] Hay tareas de ejemplo.
- [ ] Hay anuncios de ejemplo.
- [ ] Hay notas o asistencia de ejemplo.
- [ ] Hay permisos de ejemplo.
- [ ] Hay un evento de ejemplo.
- [ ] Hay registros de participantes de ejemplo.

---

## Revisión visual

- [ ] Los textos se leen bien.
- [ ] Los botones son claros.
- [ ] Las pantallas no se ven vacías.
- [ ] El diseño se ve ordenado.
- [ ] El logo o nombre del proyecto aparece correctamente.
- [ ] Los colores se ven consistentes.

---

## Preparación para la presentación

- [ ] Hay una ruta clara para la demo.
- [ ] Las cuentas de prueba están listas.
- [ ] El equipo sabe qué va a decir cada persona.
- [ ] Hay capturas de pantalla de respaldo.
- [ ] Hay un video corto de respaldo por si falla la demo.
- [ ] La computadora o celular de presentación está cargado.
- [ ] La conexión a internet fue probada.
- [ ] La presentación está lista antes del día de entrega.

---

## Resultado esperado

Al terminar la demo, el público debe entender:

- Qué problema resuelve NEXO 360.
- Quiénes pueden usar la plataforma.
- Cómo se usa la aplicación.
- Cómo funciona la parte de permisos QR.
- Cómo se conecta la página web con los eventos.
- Qué mejoras podrían agregarse en el futuro.
