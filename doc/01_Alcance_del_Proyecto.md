# 01 - Alcance del Proyecto

## Proyecto

**Nombre:** NEXO 360  
**Tipo de proyecto:** Plataforma digital escolar  
**Fecha límite:** 10 de julio  
**Nivel:** Proyecto escolar / competencia académica  

---

## Objetivo general

Crear una plataforma digital llamada **NEXO 360** que ayude a organizar en un solo lugar varios procesos escolares y de actividades juveniles, como tareas, anuncios, archivos, asistencia, notas, permisos con código QR, registros a eventos, inventarios, mapas e información para organizadores.

La idea principal es que estudiantes, docentes, personal del colegio y organizadores puedan acceder a la información que necesitan de forma más ordenada y rápida.

---

## Problema que busca resolver

Actualmente, muchas actividades escolares y eventos se organizan usando varios medios separados, por ejemplo:

- Grupos de WhatsApp.
- Hojas de papel.
- Formularios sueltos.
- Archivos enviados por diferentes canales.
- Listas hechas manualmente.
- Comunicación informal entre encargados.

Esto puede provocar:

- Pérdida de información.
- Confusión entre estudiantes y organizadores.
- Falta de control en permisos o registros.
- Dificultad para saber quién está autorizado.
- Retrasos en la comunicación.

---

## Solución propuesta

NEXO 360 propone una aplicación y un sitio web que centralicen la información más importante.

La plataforma permitiría:

- Ver tareas, anuncios y archivos escolares.
- Revisar asistencia y notas.
- Gestionar permisos estudiantiles usando códigos QR.
- Registrar personas a eventos.
- Apoyar la organización de actividades juveniles.
- Mostrar información pública de eventos mediante una página web.

---

## Usuarios principales

La plataforma está pensada para estos usuarios:

| Usuario | Uso principal |
|---|---|
| Estudiante | Consultar tareas, anuncios, archivos, notas, asistencia y permisos |
| Docente | Publicar tareas, anuncios, archivos y revisar información de sus clases |
| Personal administrativo | Supervisar permisos, estudiantes, registros y datos importantes |
| Organizador | Gestionar eventos, participantes, inventario, comunicación y mapas |
| Visitante externo | Ver información pública y registrarse a eventos desde la página web |

---

## Plataformas del proyecto

La idea completa de NEXO 360 contempla:

- Aplicación móvil Android.
- Aplicación para Windows.
- Aplicación para iOS.
- Sitio web público.

Para la primera versión, se recomienda priorizar:

1. **Android o Windows**, para mostrar la aplicación principal.
2. **Sitio web público**, para mostrar información y registros de eventos.
3. **iOS**, solo si hay tiempo suficiente para probarlo bien.

---

## Lo que debe incluir la primera versión

La primera versión no necesita tener todo perfecto ni completo. Debe enfocarse en mostrar claramente la idea del proyecto.

Debe incluir:

- Inicio de sesión.
- Diferentes accesos según el tipo de usuario.
- Portal escolar básico.
- Permisos con código QR.
- Registro a eventos.
- Panel básico para organizadores.
- Página web pública.
- Datos de prueba para la demostración.

---

## Lo que no es necesario para la primera versión

Estas funciones pueden dejarse para una versión futura:

- Chat completo en tiempo real.
- Reportes avanzados.
- Pagos en línea.
- Portal para padres.
- Mapas en tiempo real muy detallados.
- Notificaciones avanzadas.
- Estadísticas complejas.
- Versión final publicada en tiendas de aplicaciones.

---

## Resultado esperado

Al finalizar esta etapa, el equipo debe tener claro:

- Qué problema resuelve NEXO 360.
- Qué funciones sí entran en la primera versión.
- Qué funciones se dejarán para después.
- Qué usuarios usará la plataforma.
- Cómo se va a demostrar el proyecto el día de la entrega.
