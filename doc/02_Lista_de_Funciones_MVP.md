# 02 - Lista de Funciones MVP

## ¿Qué significa MVP?

MVP significa **Producto Mínimo Viable**.

En palabras simples, es la primera versión funcional del proyecto. No incluye todo lo que se podría hacer en el futuro, sino solo lo más importante para demostrar que la idea funciona.

Para NEXO 360, el MVP debe permitir enseñar la plataforma de forma clara en la competencia o presentación escolar.

---

## Funciones obligatorias para la primera versión

Estas funciones son las más importantes y deberían estar listas para la demostración.

| Función | Descripción simple | Prioridad |
|---|---|---|
| Inicio de sesión | Permite que cada usuario entre con su cuenta | Alta |
| Acceso por rol | Cada usuario ve opciones según su tipo de cuenta | Alta |
| Panel de estudiante | Muestra tareas, anuncios, archivos, asistencia, notas y permisos | Alta |
| Panel de docente | Permite ver o publicar tareas, anuncios y archivos | Alta |
| Panel administrativo | Permite revisar usuarios, permisos y registros importantes | Alta |
| Panel de organizador | Permite administrar eventos y participantes | Alta |
| Tareas | Los docentes publican tareas y los estudiantes las consultan | Alta |
| Anuncios | Se publican avisos importantes | Alta |
| Archivos | Se comparten documentos o enlaces | Media |
| Asistencia | Muestra o registra asistencia básica | Media |
| Notas | Muestra calificaciones de forma simple | Media |
| Permisos con QR | Se genera un código QR para validar permisos | Alta |
| Validación de QR | Un encargado puede escanear o verificar el permiso | Alta |
| Registro a eventos | Una persona puede registrarse a una actividad | Alta |
| Página web pública | Muestra información de eventos y permite registro | Alta |

---

## Funciones recomendadas si alcanza el tiempo

Estas funciones ayudan a que el proyecto se vea más completo, pero pueden hacerse de forma sencilla.

| Función | Versión simple recomendada |
|---|---|
| Inventario | Lista de objetos, cantidad y estado |
| Mapa o croquis | Mostrar ubicaciones principales del evento |
| Comunicación entre comités | Muro de avisos o mensajes básicos |
| Lista de participantes | Ver personas registradas o aprobadas |
| Subida de documentos | Permitir adjuntar un archivo en el registro |

---

## Funciones para una versión futura

Estas ideas son buenas, pero no son necesarias para la primera entrega.

| Función futura | Motivo para dejarla después |
|---|---|
| Chat en tiempo real | Requiere más tiempo y pruebas |
| Notificaciones automáticas | Puede complicar el desarrollo |
| Pagos en línea | Requiere seguridad adicional |
| Reportes avanzados | No es necesario para demostrar la idea |
| Portal para padres | Agrega otro tipo de usuario |
| Mapas en tiempo real | Puede ser complejo para el tiempo disponible |
| Estadísticas completas | Es mejor hacerlas cuando ya haya datos reales |
| Múltiples colegios | Es una mejora para una versión más grande |

---

## Regla para controlar el alcance

Para evitar que el proyecto se vuelva demasiado grande:

- No agregar funciones nuevas si las principales aún no funcionan.
- Si se agrega algo nuevo, debe ser simple.
- Si una función toma demasiado tiempo, se deja para después.
- La prioridad es que la demostración funcione bien.
- Es mejor presentar pocas funciones bien hechas que muchas incompletas.

---

## Lista final del MVP

La primera versión de NEXO 360 debe demostrar:

1. Un usuario puede iniciar sesión.
2. La plataforma cambia según el tipo de usuario.
3. Un estudiante puede consultar información escolar.
4. Un docente puede publicar contenido básico.
5. Un administrador puede gestionar permisos.
6. Un permiso puede validarse con código QR.
7. Una persona puede registrarse a un evento desde la página web.
8. Un organizador puede revisar registros o participantes.
9. La plataforma se entiende fácilmente durante la presentación.
